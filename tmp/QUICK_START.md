# 📝 ノート管理システム クイックスタート

新しいノート管理システムへようこそ！このガイドで、すぐに使い始められます。

## 🎯 今すぐ始める（3ステップ）

### ステップ1: 今日のノートを作成

```bash
cp tmp/notes/DAILY_NOTE_TEMPLATE.md tmp/notes/daily/$(date +%Y-%m-%d).md
```

または手動で:
```bash
cp tmp/notes/DAILY_NOTE_TEMPLATE.md tmp/notes/daily/2025-12-19.md
```

### ステップ2: 今日のノートを開いて目標を記入

```bash
code tmp/notes/daily/$(date +%Y-%m-%d).md
```

「今日の目標」セクションに今日やることを書く:
```markdown
## 今日の目標
- [ ] Flashcardコンポーネントのバグ修正
- [ ] E2Eテストの追加
- [ ] パフォーマンス測定
```

### ステップ3: 作業を開始したらログを記録

作業を始めたら、時刻とともに記録:
```markdown
### [14:30] - Flashcardコンポーネントのバグ修正
**状態**: 進行中

**内容**:
- スワイプ時のアニメーションがスムーズでない問題を調査

**メモ**:
- アニメーションのフレームレートが低い
- requestAnimationFrame を使うべき

**次のステップ**:
- react-spring の設定を調整
```

## 📊 タスク管理の使い方

### タスクトラッカーの確認

```bash
code tmp/notes/TASK_TRACKER.md
```

### タスクを追加

「現在進行中」または「次に着手」セクションにタスクを追加:

```markdown
## 📌 現在進行中 (In Progress)

| タスクID | タスク名 | 優先度 | 開始日 | 担当 | 進捗 | メモ |
|---------|---------|-------|-------|------|------|------|
| FLASH-001 | Flashcardバグ修正 | 🔴高 | 2025-12-19 | 自分 | 30% | アニメーション改善 |
```

### タスク完了時

1. 進捗を100%に更新
2. 「完了」セクションに移動
3. 日次ノートにも記録

## 🔄 日々のワークフロー

### 朝（作業開始時）
1. ✅ 今日のノートを作成
2. ✅ 今日の目標を3つ書く
3. ✅ TASK_TRACKER.md で優先タスクを確認

### 日中（作業中）
1. ✅ 作業を始めたら時刻とともにログ
2. ✅ つまづいたことや学んだことをメモ
3. ✅ 参考にしたリンクを記録

### 夕方（作業終了時）
1. ✅ 完了したタスクにチェック
2. ✅ 未完了タスクを「持ち越し」に記録
3. ✅ 明日の予定を簡単に書く

### 金曜日（週次レビュー）
1. ✅ TASK_TRACKER.md の「今週の完了タスク」を更新
2. ✅ 次週の目標を設定
3. ✅ 全体進捗を計算

## 💡 便利な Tips

### 素早く今日のノートを開く
エイリアスを設定すると便利:
```bash
alias today="code tmp/notes/daily/\$(date +%Y-%m-%d).md"
```

使用:
```bash
today
```

### 過去のノートを検索
```bash
grep -r "Flashcard" tmp/notes/daily/
```

### 今週のノートを確認
```bash
ls -lt tmp/notes/daily/ | head -7
```

## ❓ よくある質問

**Q: 毎日ノートを作らないとダメ？**
A: いいえ。必要な時だけでOK。ただし、習慣化すると便利です。

**Q: タスクIDは必須？**
A: いいえ。シンプルにタスク名だけでも大丈夫です。

**Q: 古いノートはどうする？**
A: daily/ フォルダに日付順で保存されます。検索可能です。

**Q: テンプレートをカスタマイズしてもいい？**
A: はい！自分に合った形に自由に変更してください。
