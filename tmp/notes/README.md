# ノート管理システム

このディレクトリは、プロジェクトの日々のタスク管理とノートテイキングのためのシステムです。

## 📁 ディレクトリ構造

```
tmp/notes/
├── README.md                    # このファイル
├── TASK_TRACKER.md             # シンプルなタスクトラッカー
├── DAILY_NOTE_TEMPLATE.md      # 日次ノートのテンプレート
├── daily/                      # 日次ノートの保存場所
│   └── YYYY-MM-DD.md          # 日付ごとのノート
└── archive/                    # 過去の計画やドキュメント
    └── task-decomposition-plan.md
```

## 🚀 使い方

### 1. 毎日の作業開始時

1. テンプレートをコピーして新しい日次ノートを作成:
   ```bash
   cp tmp/notes/DAILY_NOTE_TEMPLATE.md tmp/notes/daily/$(date +%Y-%m-%d).md
   ```

2. 今日の目標を記入

3. 作業を開始したら、時刻とともにログを記録

### 2. タスク管理

`TASK_TRACKER.md` を使用:

- **現在進行中**: 今作業しているタスクを記録
- **次に着手**: 優先順位をつけて次のタスクを整理
- **完了**: 週ごとに完了タスクを記録
- **ブロック中**: 問題や依存関係でブロックされているタスク

### 3. 日次ノートの書き方

各作業の開始時に以下を記録:

```markdown
### [10:30] - Flashcardコンポーネントの実装
**状態**: 進行中

**内容**:
- スワイプジェスチャーの実装
- アニメーションの追加

**メモ**:
- react-spring を使用することに決定
- パフォーマンス問題が発生、要調査

**次のステップ**:
- パフォーマンステストの実施
- E2Eテストの追加
```

### 4. 週次レビュー

毎週金曜日に:

1. `TASK_TRACKER.md` の完了タスクを確認
2. 次週の目標を設定
3. ブロックされているタスクの解決策を検討
4. 全体進捗を更新

## 💡 ベストプラクティス

### 日次ノート
- **リアルタイムで記録**: 作業後ではなく、作業中に記録
- **具体的に**: 何をしたか、なぜそうしたかを明確に
- **学びを記録**: 失敗や発見も記録
- **リンクを追加**: 参考にしたドキュメントやコードへのリンク

### タスク管理
- **優先度を明確に**: 🔴高/🟡中/🔵低 を使い分け
- **小さく分割**: 1日以内に完了できるサイズに
- **依存関係を記録**: ブロッカーを早期に特定
- **定期的に更新**: 最低でも1日1回は更新

### 週次レビュー
- **振り返りの習慣**: 何がうまくいったか、何を改善できるか
- **ベロシティの測定**: 完了タスク数を追跡
- **計画の調整**: 現実に基づいて計画を更新

## 🔧 便利なコマンド

### 今日のノートを作成
```bash
cp tmp/notes/DAILY_NOTE_TEMPLATE.md tmp/notes/daily/$(date +%Y-%m-%d).md
```

### 今日のノートを開く
```bash
code tmp/notes/daily/$(date +%Y-%m-%d).md
```

### 今週のノートを一覧
```bash
ls -lt tmp/notes/daily/ | head -7
```

### キーワード検索
```bash
grep -r "キーワード" tmp/notes/daily/
```
